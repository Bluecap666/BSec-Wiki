
# 常用 Web 代理工具

## 🎯 正向代理工具（客户端代理）

### 1. Burp Suite

 **类型** ：商业/社区版
 **特点** ：

* Web 安全测试的行业标准
* 强大的拦截、重放、扫描功能
* 丰富的插件生态系统（BApp Store）
* 支持自动化漏洞扫描

 **适用场景** ：

* 专业 Web 应用安全测试
* API 安全测试
* 渗透测试

### 2. OWASP ZAP (Zed Attack Proxy)

 **类型** ：开源免费
 **特点** ：

* OWASP 官方项目，完全免费
* 自动化主动和被动扫描
* 支持 REST API，易于集成
* 活跃的社区支持

 **适用场景** ：

* 开源项目安全测试
* CI/CD 集成
* 学习 Web 安全

### 3. Fiddler Classic

 **类型** ：免费（Windows）
 **特点** ：

* HTTP/HTTPS 流量抓包和分析
* 性能测试功能强大
* 支持断点调试和重放
* 易于使用的界面

 **适用场景** ：

* Web 开发调试
* 性能分析和优化
* 移动应用测试

### 4. Charles Proxy

 **类型** ：商业（跨平台）
 **特点** ：

* 支持 macOS、Windows、Linux
* SSL 代理配置简单
* 流量重写和断点功能
* Map Local/Remote 功能

 **适用场景** ：

* 移动应用开发调试
* API 接口测试
* 前端开发

### 5. mitmproxy

 **类型** ：开源免费
 **特点** ：

* 命令行工具，支持 Python 脚本扩展
* 透明的 HTTP/HTTPS 代理
* 支持流量录制和回放
* 可编程的拦截和修改

 **适用场景** ：

* 自动化测试
* 流量分析脚本开发
* 安全研究

## 🔄 反向代理工具

### 1. Nginx

 **特点** ：

* 高性能的反向代理服务器
* 负载均衡和缓存功能
* 灵活的配置和模块化

 **使用场景** ：

* Web 服务器负载均衡
* 静态资源缓存
* SSL 终端代理

### 2. Apache HTTP Server (mod_proxy)

 **特点** ：

* 成熟的 mod_proxy 模块
* 支持多种协议代理
* 丰富的认证和授权模块

 **使用场景** ：

* 传统 Web 应用代理
* 复杂的 URL 重写需求

### 3. Traefik

 **特点** ：

* 云原生反向代理
* 自动服务发现
* 内置 Let's Encrypt 支持

 **使用场景** ：

* 容器化环境
* 微服务架构
* Kubernetes 集群

### 4. HAProxy

 **特点** ：

* 高性能 TCP/HTTP 负载均衡
* 健康检查和故障转移
* 详细的监控指标

 **使用场景** ：

* 高流量网站的负载均衡
* TCP 应用代理

## 🛠️ 特殊用途代理工具

### 1. Proxychains

 **特点** ：

* 强制任何程序通过代理
* 支持代理链（多个代理串联）
* 透明代理支持

 **使用场景** ：

* 渗透测试中的匿名化
* 网络访问控制绕过

### 2. SOCKS 代理工具

 **工具** ：Shadowsocks、V2Ray、Trojan
 **特点** ：

* 全流量代理
* 更好的协议伪装
* 抗检测能力

 **使用场景** ：

* 网络访问加速
* 网络审查绕过

## 💡 选择建议

### 安全测试场景

* **专业测试** ：Burp Suite Professional
* **预算有限/学习** ：OWASP ZAP
* **自动化集成** ：mitmproxy + 脚本

### 开发调试场景

* **Windows 平台** ：Fiddler
* **跨平台需求** ：Charles Proxy
* **API 测试** ：Burp Suite 或 Postman

### 生产环境

* **高性能需求** ：Nginx
* **云原生环境** ：Traefik
* **传统应用** ：Apache HTTP Server

## ⚠️ 使用注意事项

1. **法律合规** ：确保在授权范围内使用代理工具
2. **证书安装** ：HTTPS 代理需要安装根证书
3. **性能影响** ：代理可能引入延迟，生产环境谨慎使用
4. **安全风险** ：代理工具可能成为攻击入口，做好安全防护
